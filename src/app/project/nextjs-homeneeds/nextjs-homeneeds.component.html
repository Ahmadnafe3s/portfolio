NextJS Home Needs

Nextjs_HomeNeeds is a web application that is the advanced version of my HomeNeeds project, which I originally created using Angular. The Next.js HomeNeeds project is designed for posting recipes and managing shopping lists, allowing users to view posted recipes, post their own recipes to the world, and perform CRUD operations on their own posts. Leveraging technologies like Bootstrap for styling and Next.js for both front-end design and back-end functionality, and using MongoDB Atlas for data storage, this project is divided into several sections, including Authentication, Recipes, Shopping List, and Profile.

Challenges Faced

  • I recently encountered a couple of issues. First, I wanted to retrieve searchParamsData (query params) in the route (back-end) that I passed with the API. I tried various methods but couldn't get the searchParamsData. I searched extensively on Google, Stack Overflow, and several other places, but I couldn't find a solution. Finally, after consulting ChatGPT, I found the solution: I needed to simply extract the searchParams from the new URL(req.url) object.
  • The second issue I faced was passing user data into the navbar from the login component. These components were quite far from each other, so it required a lot of props drilling, which could cause performance issues in the application. I overcame this by using Redux Toolkit.
  • The third issue I faced was that after passing the data into the navbar, the data in Redux Toolkit would get wiped whenever I refreshed the page. To overcome this problem, I stored the data in localStorage by dispatching it from the login component.
  • The fourth issue I faced was that after storing data in localStorage, whenever the user clicked the logout button, they were immediately logged out from the application. Behind the scenes, the application removed the localStorage data and token. However, if the user didn't click the logout button, they were automatically logged out after 24 hours due to token expiration, as the token is automatically removed from the cookies after expiration. The problem was that the data in localStorage still remained even after logout. To overcome this problem, I used cookies to store user data instead of localStorage to persist RTK store data. I set the cookie expiration time to be the same as the token expiration, ensuring that user data is automatically deleted after the token expires.
